## V15 — 요청형 시각 생성

### Upgrade from V14
V0~V14 = 시각이 자동으로 생성됨 (도구 호출 → 시각).
V15 = **사용자가 자연어로 요청** → 그 시각 즉시 생성.

### Why (본인 비전 매핑)
> 사용자가 본인 페인에 정확히 맞춘 시각이 필요할 때. "지난 1시간 동안 결제 흐름 보여줘" 같은.

### Exit criteria
- 입력 박스에 자연어 요청
- LLM이 요청 분석 → 시각 명세 생성
- 즉시 렌더링
- 저장 가능 ("이 시각 즐겨찾기")

### 구체 시각화 예시

```
┌─ 시각 요청 ────────────────────────────────┐
│ 무엇을 보고 싶으세요?                        │
│ [지난 1시간 동안 api.py 가 어떻게 변했나]   │
│ [생성]                                      │
└─────────────────────────────────────────────┘
       ↓ LLM 처리 3초
       ↓
┌─ 결과 시각 ────────────────────────────────┐
│ api.py 변화 (1시간)                         │
│                                            │
│ 13:42 - 첫 수정 (verify_password 추가)     │
│ 14:01 - 리팩터링 (3개 함수 분할)            │
│ 14:23 - 테스트 추가                         │
│ 14:55 - 버그 수정                           │
│                                            │
│ [총 4번 수정, 100줄 → 145줄]                │
└─────────────────────────────────────────────┘
```

### Architecture
- 자연어 입력 → LLM 분석 → 시각 명세 (V4d 재활용)
- 과거 데이터 retrieval (timeline 검색)
- 클라이언트: 시각 컴포넌트 동적 생성

### Dependencies
- V4d (LLM 자유 시각 명세)
- 과거 이벤트 저장소 (현재는 메모리 500개. 더 큰 저장 필요)

### Risk
- LLM 이해 실패
- 요청 모호하면 빈약한 시각
- 과거 데이터 양 한계

### 본인 비전 적합도
- ★★★★★ — 본인 비전 핵심: "사용자가 옵션 줄 때 그림으로 비교"의 풀 버전. 본인이 직접 시각 요청 가능.
