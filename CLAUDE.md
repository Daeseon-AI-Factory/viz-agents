# CLAUDE.md — VIZ 제품 작업 컨텍스트

> 이 폴더에서 작업할 때 AI가 **반드시 먼저 읽고 시작**해야 하는 문서.
> 이 문서의 핵심 목적과 사용자 프로필을 절대 잊지 말 것.

---

## 핵심 목적 (NEVER FORGET)

> **"코드, 시스템, 제품, 아키텍처, 비즈니스를 전부 시각화해서 애니메이션처럼 보여주고,
> 사용자가 텍스트 wall을 읽지 않고 바로 이해하게 한다."**

- 사용자 본인이 비-엔지니어 / 영어 약함 / 집중력 짧음
- 본인이 본인 같은 사람을 위해 만드는 제품
- 시장 가설이 아니라 **본인의 실제 페인** 해결이 동기
- AI가 옵션 줄 때도 **그림으로 비교**되어야 함 (텍스트로 풀어 설명하는 거 금지)

---

## 사용자 프로필 (모든 응답에 적용)

- 비-엔지니어, 한국어 사용 — 한국어로 응답
- 영어 약함 — 영어 용어 최소화, 쓸 거면 짧게 병기
- 집중력 짧음 — **응답은 짧게**. 표·박스·구분선으로 시각화. 결론부터.
- 텍스트 wall 못 읽음 — 줄글 단락 금지. 시각 구조 필수.
- AI 옵션 비교 못함 — 옵션 제시 시 `AskUserQuestion`의 `preview` 적극 활용해 시각 비교 가능하게.
- 깊이 들어가기 전에 **한 단계씩 동의 받고** 진행.

## viz-v0~v6 파일의 정체 (사용자 명시 — 정정본)

사용자 발언 (2026-05-22):
> "버전 파일들은 제품의 업그레이드 단계인거다. 최종 제품은 계속 고도화할거다.
>  학습 목적으로 버전들을 나눈거다. 버전에 따라서 단계적인 프로젝트 고도화를 설명하려는거다."

해석:
- v0~v6는 **실제 제품 업그레이드 단계들**. (자기설명 메타구조 아님 — 그건 내 오해였음)
- **최종 제품은 계속 고도화 / 종착점 없음.**
- 버전을 나눈 이유는 **학습 목적** — 단계적 고도화를 보여주기 위함.
- 즉: 빌드 마일스톤 + 교육 콘텐츠. 두 역할 동시.
- 현재 v0~v6 내용은 엔지니어 도구 색채(call graph, AST) → 진짜 비전(동적 시각화)과 갭. **점진적 재정렬 필요** (한 번에 갈아엎지 말고 단계적으로).

## Companion Window 결정 (2026-05-22)

폼팩터: **브라우저 탭** (로컬 웹서버) — Streamlit 또는 Next.js.

```
[터미널]              [크롬 탭]
 Claude Code   |  localhost:8501
               |  ┌── VIZ ──┐
               |  │ 애니 재생 │
               |  └────────┘
```

동작 원칙:
- Claude Code(브레인) ↔ 로컬 웹서버(캔버스)
- 같은 폴더의 `viz_spec.json` 변경 감지 → 브라우저 자동 새로고침
- 사용자는 채팅 텍스트 안 읽고 **옆 창만 봐도 이해**
- 머신 바꿔도 안정 / URL 공유 가능 / 의존 가벼움

이유: VS Code 의존성, Electron 개발 부담 회피. 가장 호환성·확장성 좋음.

## 제품 경계

### DOES (이 4가지만)
| 모드 | 한 줄 |
|---|---|
| ① Comprehension | "이게 뭐 하는 건지" → 애니메이션으로 |
| ② Decision | "AI 옵션 3개 비교" → 시각 카드로 |
| ③ Live | "지금 돌고 있는 거" → 실시간 트레이스 |
| ④ What-if | "바꾸면 어떻게 돼" → 시뮬레이션 |

### DOES NOT
- 코드 작성 (SI 담당)
- 테스트 실행 (QA 담당)
- 운영 (OPS 담당)
- 결정 자체 — 옵션 생성은 도메인 에이전트 / VIZ는 **보여주기만**
- 긴 텍스트 문서 (DOC 담당)

### MODES
- **Standalone** — 파일/repo 던지면 시각화
- **Integrated (universal HUD)** — 다른 에이전트(SI/QA/OPS/...) 출력이 항상 VIZ를 거쳐서 사용자에게

## 플랫폼 컨텍스트

aicore = 8개 AI 직군 서비스(SI/QA/OPS/VIZ/CORE/ADR/DOC/AUTO).
VIZ는 4번. 다른 서비스 출력의 universal visual layer 역할.

폴더:
- `aicore/ai-core/` — 실제 구현 모노레포
- `aicore/dev-agents/` — SI 빌드 스펙
- `aicore/viz-agents/` — VIZ 빌드 스펙 (현재 폴더)

## 즉시 목표 (2026-05-22 사용자 명시)

> "브라우저에서 클로드코드 터미널을 열고 일을 시키는데, 이게 시각적으로 꼭 명확히 보고싶다.
>  비용은 더 쓰겠지만 상관없다."

**V0 타겟 = "Claude Code Live HUD"**:
- 사용자는 브라우저에서 Claude Code(claude.ai/code 또는 로컬) 사용
- **옆 브라우저 탭**에 VIZ HUD 항상 켜둠
- Claude Code의 모든 액션(파일 읽기/수정, Bash, 서브에이전트 spawn, 결정)이 **실시간으로 시각화**
- 사용자는 채팅창 텍스트 안 읽고 옆 탭만 봐도 무슨 일 벌어지는지 즉시 파악

연결 방식: Claude Code **hooks** (settings.json) → 로컬 웹서버 → WebSocket → 브라우저 탭.

비용 정책: **무시**. 정확성·즉시성 우선.

## 작업 규칙 (Working Rules)

1. **응답은 짧게**. 줄글 단락 쓰지 말 것. 표/박스/구분선/리스트.
2. **결론부터**. 근거는 뒤에 간략히.
3. **옵션 줄 때**: `AskUserQuestion` + `preview` 사용 → 시각 비교 가능하게.
4. **큰 결정 전**: 한 단계씩 사용자 동의 받기.
5. **새로 알게 된 핵심 사실**: 메모리(`~/.claude/projects/.../memory/`) 또는 이 CLAUDE.md에 박기.
6. **사용자가 지칠 신호**: "ㄴ", "ok", "그래" 같은 짧은 응답 → 더 짧고 시각적으로.
