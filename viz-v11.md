## V11 — What-If 시뮬레이터

### Upgrade from V10
V10 = 일어난 일의 애니메이션.
V11 = **아직 안 일어난 일**의 시뮬레이션. "이거 적용하면 어떻게 될까?"

### Why (본인 비전 매핑)
> 본인이 옵션 선택할 때 — "AI가 옵션 3개 줬는데 그림으로 비교" 발전형. 각 옵션의 **결과**를 미리 시뮬레이션.

### Exit criteria
- Claude 가 코드/시스템 변경 제안 시 → 적용 전 시뮬레이션 시각
- 비교: "현재 vs 변경 후" 나란히
- 비즈니스 메트릭 예측 변화
- 시스템 부하 변화

### 구체 시각화 예시

```
┌─ 변경 제안: "결제 API timeout 5s → 30s" ────────┐
│                                                  │
│  현재                  변경 후                    │
│  ─────────             ─────────                  │
│  p95: 320ms           p95: 380ms                  │
│  실패율: 2.3%          실패율: 0.8% ✓              │
│  타임아웃: 12/일       타임아웃: 2/일 ✓            │
│  DB 연결 풀: 안전     DB 연결 풀: ⚠️ 부족 가능    │
│                                                  │
│  💡 추천: 적용 + DB 연결 풀 늘리기 동시          │
│                                                  │
│  [Apply] [Cancel] [What if also...]              │
└──────────────────────────────────────────────────┘
```

### Architecture
- LLM: 변경 → 영향 예측
- 외부 데이터 (선택): 현재 메트릭 (Datadog, Grafana 등)
- 시뮬레이션 엔진 (선택): 간단한 큐 모델

### Dependencies
- LLM (높은 추론 비용)
- 외부 메트릭 API (선택)

### Risk
- 시뮬레이션 정확도 (LLM 추측)
- "수정 후 데이터" 가짜일 수 있음 → 신뢰성 표시 필수
- 과한 자신감 → 잘못된 결정 유도

### V9 와의 차이
- V9 = 현재 KPI 시각
- V11 = 미래 가설 KPI 시뮬레이션
