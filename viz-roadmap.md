## V4~V20 한 줄 요약

| V | 한 줄 | 본인 비전 영역 |
|---|---|---|
| **V4** | LLM-Driven Dynamic Visualization — 작업 디스크립션에 따라 시각이 그때그때 다르게 생성 | 동적 + 코드/시스템 표면 |
| **V5** | Code Deep Viz — AST 차원의 변화, 함수 시그니처 변경, 의존성 변화 시각 | 코드 깊이 |
| **V6** | System Topology Live Map — 서비스/모듈 네트워크 도식 + 실시간 데이터 흐름 | 시스템 |
| **V7** | User Journey Visualization — 사용자 의도 → 화면/액션/결과의 흐름 | 제품 |
| **V8** | Architecture Layer Map — 레이어/컴포넌트/배포 시각 + 트래픽 오버레이 | 아키텍처 |
| **V9** | Business KPI Stream — 가치사슬, 돈 흐름, KPI 실시간 시각 | 비즈니스 |
| **V10** | Real Animation Engine — 정적 X. CSS/JS 진짜 움직이는 애니메이션 | 애니메이션 ★ |
| **V11** | What-If Simulator — "이 변경 적용하면?" 시뮬레이션 시각 | 의사결정 보조 |
| **V12** | Interactive Drill-Down — 시각 위 클릭 → 깊이 들어가서 더 본다 | UX 깊이 |
| **V13** | Multi-Agent Integration — 다른 AI 에이전트(SI/QA/OPS/...) 출력 통합 HUD | 플랫폼 |
| **V14** | Multi-Pane Layout — 화면 분할, 여러 시각 동시 비교 | UX 광활 |
| **V15** | On-Demand Viz Generation — 사용자가 "이걸 보여줘" 자연어 → 시각 자동 생성 | 인터페이스 |
| **V16** | Collaboration — 여러 사람이 같은 HUD 공유 (페어 디버깅) | 팀 |
| **V17** | Mobile Adaptive — 모바일/태블릿 적응 레이아웃 | 휴대성 |
| **V18** | Export & Share — 시각 캡처, 공유 링크, 외부 임베드 | 산출물 |
| **V19** | Time-Travel Scrub — 과거 시점으로 시간축 스크럽, 재생 | 회고/디버깅 |
| **V20** | Self-Learning Viz — 사용자 피드백("이 시각 좋아/별로")으로 LLM이 viz 품질 자가 개선 | 진화 |

## 합리적 빌드 순서 (본인 비전 기준)

가장 본인 비전에 직격하는 순서:

```
즉시 빌드 (V4):
  ↓
V4 a → b → c → d  (LLM-driven dynamic. 코드 빌드 완료)

본인 평가 후 (V5-V10):
  ↓
V5 (코드) → V6 (시스템) → V7 (제품) → V8 (아키텍처) → V9 (비즈니스)
  → V10 (애니메이션) — 6대 비전 영역 차례로
  
선택적 (V11-V20):
  ↓
V11 (what-if), V12 (drill), V13 (multi-agent), V14 (multi-pane),
V15 (on-demand), V16 (collab), V17 (mobile), V18 (export),
V19 (time-travel), V20 (self-learning)
```

V5~V10 = 본인 비전 6영역 직격. **이 6개가 풀로 가야 본인이 처음 말한 비전 90% 달성.**
V11~V20 = 부가 가치. 본인 진짜 페인이 어디인지 V5-V10 빌드하면서 드러남.

## 위험 / 솔직 노트

- 각 버전이 LLM 호출 비용 추가 가능 (V4~V10 거의 다). 본인 OK 했지만 합치면 월 $10-50 가능.
- V5~V20는 spec만 작성. **실제 코드 빌드는 본인 평가 후 결정.**
- V10 (진짜 애니메이션)이 가장 임팩트 클 듯 — 사용자 처음 비전 단어 "애니메이션처럼" 직격.
- 각 버전 spec은 V0~V6 (기존 viz-v0.md~viz-v6.md) 형식 따라가되 HUD 컨텍스트로 재해석.
