## V19 — 시간축 스크럽 (재생)

### Upgrade from V18
V0~V18 = 현재 시각.
V19 = **과거 시점**으로 시간축 스크럽 + 재생. 마치 비디오 plays / Apple Time Machine.

### Why
- 디버깅: "에러 발생한 그 순간 뭐 했나?"
- 회고: "어제 이 기능 어떻게 만들었지?"
- 학습: 다른 사람 작업 시간축 재생

### Exit criteria
- 시간축 슬라이더 (오늘 / 어제 / 이번 주)
- 임의 시점으로 점프
- 재생/일시정지/속도 조절 (1x, 2x, 5x)
- 키프레임 (UserPromptSubmit 같은 의미 있는 순간 마커)

### 구체 시각화 예시

```
┌─ Time Travel ──────────────────────────────────┐
│                                                │
│ [▶] [⏸] [⏭] 속도: [1x ▼]                       │
│                                                │
│ ◆━━━━●━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━◆       │
│ 13:00  현재 13:42                  18:00       │
│                                                │
│  ◆ = 의미 있는 순간 (UserPromptSubmit)         │
│                                                │
│ 현재 시점: 13:42:15                            │
│ 무엇을 보고있나: api.py Edit (verify_password) │
│                                                │
└────────────────────────────────────────────────┘
```

### Architecture
- 모든 이벤트 영구 저장 (현재 메모리 500 → 디스크 DB)
- 클라이언트: 시간축 슬라이더 + 시각 컴포넌트 시점별 rerender
- LLM (선택): 시간 범위 자동 요약

### Dependencies
- 백엔드: SQLite 또는 DuckDB (이벤트 저장)
- frontend: 슬라이더 컴포넌트

### Risk
- 데이터 양 (하루 1000+ 이벤트 = 한 달 30000+)
- 시각 컴포넌트 시점별 렌더링 비용
- UX (시간축 너무 길면 정밀 조작 어려움)

### V10 (애니메이션) 와의 관계
- V10 = 미래/현재 일어나는 일의 애니메이션
- V19 = 과거 일어난 일의 재생 (역 애니메이션)
- 둘 다 합치면 = 풀 비디오 플레이어 같은 UX
