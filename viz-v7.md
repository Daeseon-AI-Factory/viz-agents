## V7 — 사용자 여정 시각 (Product Visualization)

### Upgrade from V6
V6 = 시스템 내부 구조.
V7 = **사용자가 제품을 어떻게 쓰는지** 시각화. 페르소나 → 화면 → 액션 → 결과 흐름.

### Why (본인 비전 매핑)
> "제품 시각화" — Claude가 제품 기능 개발하면, 그 기능이 사용자 입장에서 어떻게 보일지 자동 시각.

### Exit criteria
- 사용자 여정 다이어그램 (entry → ... → goal)
- 각 단계에서 어떤 화면/API/액션이 일어나는지
- 새 기능 추가 시 여정 어디에 끼는지 강조
- A/B 분기 표현

### 구체 시각화 예시

```
┌─ 사용자 여정: 로그인 → 결제 → 사용 ─────────────────┐
│                                                     │
│  👤 첫 방문                                          │
│      ↓                                              │
│  [회원가입 화면]                                     │
│      ↓ (3.2초)                                      │
│  [이메일 인증] ⚠️ 12% 이탈                          │
│      ↓                                              │
│  [결제 화면]                                         │
│      ↓ (← Claude가 방금 여기 수정)                   │
│  [구독 활성화]                                       │
│      ↓                                              │
│  🎯 첫 사용                                          │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Architecture
- 프로젝트의 라우터/페이지/엔드포인트 자동 추출
- LLM: "이 코드가 사용자 여정의 어디에 속하나?" 분류
- 클라이언트: 가로 플로우 차트 + 단계별 박스

### Dependencies
- frontend: 그래프 라이브러리 (V6 재활용)
- backend: 라우터 추출 (FastAPI: `app.get/post`, Next.js: `pages/`)
- LLM: 여정 단계 매핑

### Risk
- "사용자 여정" 정의가 프로젝트마다 다름
- 정확한 단계 식별이 어려움 (LLM 추측 의존)
- 백엔드만 있는 프로젝트는 여정 추출 어려움

### V8 (아키텍처) 와의 차이
- V7 = 사용자 입장 (페이지/액션 흐름)
- V8 = 시스템 입장 (컴포넌트/배포)

### V9 (비즈니스) 와의 차이
- V7 = 어떻게 쓰나 (행동)
- V9 = 얼마나 가치 (돈/KPI)
